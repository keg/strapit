What's included

Strapit is downloadable in two forms, within which you'll find the following directories and files, logically grouping common resources and providing both compiled and minified variations.

jQuery required

Please note that all JavaScript plugins require jQuery to be included, as shown in the starter template.

Precompiled Strapit

Once downloaded, unzip the compressed folder to see the structure of (the compiled) Strapit. You'll see something like this:

{% highlight bash %} strapit/ ├── css/ │ ├── strapit.css │ ├── strapit.min.css ├── js/ │ ├── strapit.js │ └── strapit.min.js └── fonts/ │ ├── glyphicons-halflings-regular.eot │ ├── glyphicons-halflings-regular.svg │ ├── glyphicons-halflings-regular.ttf │ └── glyphicons-halflings-regular.woff └── img/ ├── webicons/ {% endhighlight %}

This is the most basic form of Strapit: compiled files for quick drop-in usage in nearly any web project. We provide compiled CSS and JS, as well as compiled and minified CSS and JS. Fonts from Glyphicons are included and images from Fairfield Creative's FC Webicons.

Strapit source code

The Strapit source code download includes the precompiled CSS, JavaScript, font and image assets, along with source Less, JavaScript, and documentation. More specifically, it includes the following and more:

{% highlight bash %} strapit/ ├── less/ ├── js/ ├── fonts/ ├── dist/ │ ├── css/ │ ├── js/ │ └── fonts/ │ └── img/ │ └──── webicons/ └── docs/ └── examples/ {% endhighlight %}

The less/, js/, and fonts/, and img/ are the source code for our CSS, JS, icon fonts and social web icons (respectively). The dist/ folder includes everything listed in the precompiled download section above. The docs/ folder includes the source code for our documentation, and examples/ of Strapit usage. Beyond that, any other included file provides support for packages, license information, and development.

Less source code structure

If you download the source code, you'll need to know something about the structure of the Less files within.

Unzip the compressed folder of the source code and look within the folder named less to see Strapit's pre-complied structure.

{% highlight bash %} less/ ├── custom/ │ ├── *.less ├── framework/ │ ├── *.less ├── prop/ │ ├── application.js │ └── variables.js └── vendor/ │ ├── glyphicons.less │ └── webicons.less | └── style.less {% endhighlight %}

The custom folder contains all custom style files built exclusively for DFM.

The framework folder contains all the base style files including the CSS reset, grid, typography and UI objects that come from Bootstrap and Foundation.

The prop folder contains two files application.less and variables.less. This folder is where the custom files for your site should go. It also includes the variables file that will determine the color palette, spacing and typography choices for your site.

The application.less file serves as a complier for the custom Less files you create &emdash; similar to the style.less file contained in the root folder. When you open style.less, you will notice that application.less is one of the lasts file within the complication chain. style.less is also where you can comment out the styles you may not need from the master framework files.

Finally, the vendor folder should contain any third-party plugin style files you want to add into your project. As an example, the glyphicons.less and webicons.less files are contained here.

Less compilation

If you download the original files, you need to compile Strapit's Less files into usable CSS. To do that, you will need a complier. Here are some options:

Third party compliers

Compliers run alongside your browser and recompile your Less files per save. For Mac users, CodeKit or LiveReload are good options. If you have a Windows machine, SimpLess is an option (it's also available for Mac users) and so is WinLess.

Command-line compliers

If you are comfortable with the command-line, then some options for you are Less Watcher (requires npm), Watchr (requires Ruby Gems), or you can do it by hand (scroll down to Server-side usage).

Client-side usage

We do not recommend this for many, many reasons, but it's an option. Read the documentation on how to do it this way.
